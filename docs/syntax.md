# 基本構文

このプロジェクトで作る Markdown 系言語の最初の仕様です。まずは
コンパイラ、シンタックスハイライト、フォーマッタで共有できる構文だけを
定義し、パーサーの実装時にこの仕様を実行例とテストへ落とし込みます。

## 強調

`*` で囲んだ文字列を太字として扱います。

```md
これは *重要な文章* です。
```

```html
これは <strong>重要な文章</strong> です。
```

斜体は提供しません。したがって、`_文章_` や `*文章*` を斜体として
解釈することはありません。`*文章*` は常に太字です。

太字の範囲内に未対応の入れ子構文を持ち込まず、まずは単純なインライン
テキストを対象にします。アスタリスクを文字として表示したい場合の
エスケープ方法は、パーサー実装時に別途定義します。

## リスト

行頭の `+` は番号付きリストを表します。番号はソースに書かず、出力時に
`1` から連番で生成します。

```md
+ 最初の項目
+ 次の項目
+ 最後の項目
```

```html
<ol>
  <li>最初の項目</li>
  <li>次の項目</li>
  <li>最後の項目</li>
</ol>
```

`+` から始まる連続した行は同じ番号付きリストにまとめます。空行または
別のブロック要素でリストを終了します。

通常の箇条書きには行頭の `-` を使います。

```md
- 選択肢 A
- 選択肢 B
```

`*` を行頭の箇条書き記号としては扱いません。これにより、太字記法との
役割を明確にします。
